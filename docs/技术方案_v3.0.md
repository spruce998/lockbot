## 1. 项目概述

### 1.1 项目背景

洛克王国世界（Roco Kingdom: World）是腾讯魔方工作室群于 **2026年3月26日** 全平台上线的精灵大世界游戏。基于"洛克王国"IP，集精灵收集对战、开放世界探索与自由轻松社交于一体。

**PVP 对战特点**：
- 回合制战斗，每回合从 4 个技能中选择 1 个
- **18 种属性**，存在复杂克制关系
- **能量系统**（非 PP 制），技能能耗 0-7 不等
- **技能位置效果**，技能在不同栏位有不同效果
- 特性系统、状态叠加、蓄力/迸发/传动等机制

### 1.2 项目目标

| 阶段 | 目标 | 交付周期 |
|------|------|---------|
| **Phase 1 (MVP)** | 辅助工具：识别对手 + 技能推荐 | 1-2 周 |
| **Phase 2** | 半自动：自动点击技能 + 反检测增强 | 2-4 周 |
| **Phase 3** | 全自动：PVP + 跑图 + 资源采集 | 4-8 周 |
| **Phase 4+** | 高级 AI 决策、多账号管理、云端同步 | 8-16 周 |

### 1.3 核心设计原则

| 原则 | 说明 |
|------|------|
| **模块化** | 各功能模块独立，可插拔 |
| **数据驱动** | 所有宠物/技能/属性数据从 BWIKI 自动采集 |
| **安全性** | 图像识别方案，不修改游戏内存/协议 |
| **可配置** | 所有参数通过配置调整，无需改代码 |
| **可观测** | 完善的日志、监控、统计功能 |

#### 2.2.2 爬虫流程

```
1. 通过 MediaWiki API 获取宠物列表
   action=query&list=categorymembers&cmtitle=Category:精灵&cmlimit=500
   ↓
2. 逐个抓取宠物详情页 (DOM 解析)
   - 属性: div.rocom_sprite_grament_attributes
   - 种族值: div.rocom_sprite_info_title
   - 六维: div.rocom_sprite_info_qualification
   - 特性: div.rocom_sprite_info_characteristic_content
   - 技能: div.rocom_sprite_skill_box × N
   - 描述: div.rocom_sprite_info_content
   ↓
3. 技能详情页解析（补充属性信息）
   ↓
4. 属性克制表解析（克制计算器页面）
   ↓
5. 数据导出 JSON + 导入 SQLite
```

#### 6.1.2 决策引擎（基于真实游戏机制）

```python
@dataclass
class BattleContext:
    """战斗上下文 - 基于真实游戏机制"""
    battle_id: str
    timestamp: datetime
    
    # 我方状态
    my_pet: Pet                     # 宠物信息（含种族值、特性）
    my_hp: int                      # 当前血量
    my_max_hp: int                  # 最大血量
    my_energy: int                  # 当前能量
    my_status_effects: list         # 增益/减益状态
    my_skill_positions: dict        # 技能栏位 {1: skill, 2: skill, ...}
    my_battle_log: list             # 本回合已使用技能历史
    my_buffs: dict                  # 永久加成（如过曝叠加）
    
    # 敌方状态
    enemy_pet: Pet                  # 敌方宠物
    enemy_hp_percent: float         # 血量百分比
    enemy_energy: int               # 敌方能量（推测）
    enemy_status_effects: list      # 敌方状态
    enemy_last_skill: str           # 敌方上次使用技能
    
    # 战场状态
    weather: str                    # 天气效果
    field_effect: str               # 场地效果
    turn_number: int                # 回合数
    is_my_turn: bool                # 是否我方回合
    
    # 队伍信息（用于传动/奉献）
    my_team: list                   # 队伍宠物列表
    team_energy: list               # 队伍能量状态


class DecisionEngine:
    """
    基于洛克王国世界真实机制的决策引擎
    
    策略可配置：
    - greedy: 贪心（期望能量利用率最高）
    - minimax: Minimax + Alpha-Beta（Phase 2）
    - mcts: 蒙特卡洛树搜索（Phase 3）
    """
    
    def decide(self, ctx: BattleContext) -> Decision:
        """做出决策"""
        best_skill = None
        best_score = -inf
        
        for skill in ctx.my_skill_positions.values():
            score = self._evaluate_skill(skill, ctx)
            if score > best_score:
                best_score = score
                best_skill = skill
        
        return Decision(
            skill=best_skill,
            score=best_score,
            reason=self._explain_decision(best_skill, ctx)
        )
    
    def _evaluate_skill(self, skill, ctx) -> float:
        """评估技能价值"""
        # 1. 能量利用效率
        if skill.energy_cost > ctx.my_energy:
            return -inf  # 能量不足，不可用
        
        energy_efficiency = skill.power / max(skill.energy_cost, 1)
        
        # 2. 属性克制倍率
        type_mult = get_type_multiplier(skill.element, ctx.enemy_pet.element)
        
        # 3. 位置效果加成
        position_bonus = self._calc_position_bonus(skill, ctx)
        
        # 4. 迸发/蓄力加成
        burst_bonus = self._calc_burst_bonus(skill, ctx)
        
        # 5. 状态影响
        status_modifier = self._calc_status_modifier(ctx)
        
        # 6. 敌方能量推测
        enemy_energy_factor = self._predict_enemy_energy(ctx)
        
        # 7. 血量因素（低血量时优先保命/高伤害）
        hp_factor = 1.0
        if ctx.my_hp / ctx.my_max_hp < 0.3:
            if skill.damage_type == '状态' and '回复' in skill.effect_description:
                hp_factor = 2.0  # 低血量优先回复
        
        # 8. 永久加成追踪
        permanent_bonus = self._calc_permanal_bonus(skill, ctx)
        
        score = (energy_efficiency * type_mult * status_modifier * hp_factor
                 + position_bonus + burst_bonus + permanent_bonus)
        
        return score
```

#### 6.1.3 能量管理策略

```
┌─────────────────────────────────────────────────────────────┐
│                   能量管理策略                                 │
│                                                             │
│  基本原则：                                                   │
│  1. 保持能量在 2-4 范围内（可灵活使用大多数技能）             │
│  2. 避免能量溢出（浪费回复能量）                              │
│  3. 关键时刻攒高能耗技能                                      │
│                                                             │
│  决策树：                                                     │
│  if 敌方血量 < 20%:                                          │
│      → 优先高伤害技能终结                                     │
│  elif 我方能量 >= 5:                                         │
│      → 使用高能耗技能（3-7能耗）                              │
│  elif 我方能量 <= 1:                                         │
│      → 使用低能耗/0能耗技能 或 能量回复技能                   │
│  elif 我方受到持续伤害:                                      │
│      → 优先解除状态 或 回复技能                               │
│  elif 敌方有增益:                                            │
```

### 6.3 战斗监控模块

```python
class BattleMonitor:
    """
    战斗监控 - 实时跟踪战斗状态
    
    需要识别的 UI 元素：
    1. 我方/敌方血条 → 血量百分比
    2. 我方/敌方宠物名称/图标 → 宠物识别
    3. 技能栏 4 个按钮 → 当前可用技能
    4. 能量条 → 当前能量值
    5. 状态图标 → 当前 buff/debuff
    6. 天气/场地效果图标
    7. 回合提示
    """
    
    def detect_battle_state(self, frame) -> BattleContext:
        """从截图中检测战斗状态"""
        ctx = BattleContext()
        
        # 1. 检测是否在战斗中
        ctx.in_battle = self._detect_battle_ui(frame)
        if not ctx.in_battle:
            return ctx
        
        # 2. 识别敌方宠物
        ctx.enemy_pet = self._identify_enemy_pet(frame)
        
        # 3. 识别血量
        ctx.my_hp = self._detect_my_hp(frame)
        ctx.enemy_hp_percent = self._detect_enemy_hp(frame)
        
        # 4. 识别能量
        ctx.my_energy = self._detect_energy(frame)
        
        # 5. 识别状态效果
        ctx.my_status_effects = self._detect_status_effects(frame)
        
        # 6. 识别技能栏
        ctx.my_skill_positions = self._detect_skill_slots(frame)
        
        return ctx
```

## 7. 反检测系统设计

### 7.1 7 层防封策略

| 层级 | 策略 | 实现 | 效果 |
|------|------|------|------|
| **1** | 请求限速 | 2-5 秒随机延迟 | 避免固定频率被检测 |
| **2** | UA 轮换 | 每 50 次请求切换 UA | 模拟不同浏览器 |
| **3** | 会话管理 | 保持 Cookie 连接 | 模拟正常浏览行为 |
| **4** | 指数退避 | 429/503/403 自动重试 | 遇到限速自动减速 |
| **5** | 断点续传 | checkpoint 记录进度 | 中断后可恢复 |
| **6** | 分批休息 | 每 20 个请求休息 10-30 秒 | 模拟人类休息 |
| **7** | 时间分散 | 活跃时间 8:00-23:00 | 避免深夜异常行为 |

```python
class AntiDetect:
    """操作层反检测"""
    
    def human_click(self, x, y):
        """拟人化点击"""
        # 1. 随机延迟
        time.sleep(random.uniform(0.1, 0.5))
        
        # 2. 贝塞尔曲线移动鼠标
        self._smooth_move_to(x, y)
        
        # 3. 点击位置微偏移（±8 像素）
        offset_x = random.randint(-8, 8)
        offset_y = random.randint(-8, 8)
        pyautogui.click(x + offset_x, y + offset_y)
        
        # 4. 点击后随机延迟
        time.sleep(random.uniform(0.05, 0.15))
    
    def random_idle(self):
        """随机空闲（模拟思考）"""
        time.sleep(random.uniform(1, 5))
    
    def should_rest(self) -> bool:
        """判断是否需要休息"""
        elapsed = (datetime.now() - self.session_start).seconds
        if elapsed > random.randint(3600, 7200):  # 1-2 小时
            return True
        return False
```
